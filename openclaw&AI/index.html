怎么安装？

一句话：一条命令搞定

🪟 Windows

winget install openclaw

🍎 macOS

brew install openclaw

🐧 Linux

curl -fsSL https://get.openclaw.ai | bash

📦 Docker

docker run -d --name openclaw openclaw/openclaw

配置与安全

🔑 添加 API Key

openclaw config set provider openai key sk-xxx
openclaw config set provider deepseek key sk-xxx

✅ 快速验证

openclaw --version      # 确认 CLI 可用
openclaw doctor         # 检查配置问题
openclaw gateway status # 确认服务运行

🔐 安全要点

  • 所有数据本地存储
  • 敏感信息支持 SecretRef 加密存储
  • 支持 OAuth 认证 + 细粒度权限控制

快速使用

🌐 控制台 UI

访问 http://127.0.0.1:18789

  • Vite + Lit 构建的单页应用
  • 直接在浏览器上对话
  • 可视化配置管理

📱 连接外部平台

openclaw channels add --channel qqbot --token "***"
openclaw channels add --channel discord --token "***"
openclaw channels add --channel telegram --token "***"

> 写一次 Skill，QQ / Discord / Web 同步可用

Hermes

让 Agent 拥有「手」和「眼睛」

Hermes 是什么？

Agent 执行引擎 — 让 AI 不只「会说」，更能「会做」

核心能力

  • 🧩 工具调用 — 调 API、读文件、写代码
  • 👀 视觉识别 — 看懂截图、UI、流程图
  • 🖐️ 浏览器操控 — AI 自动操作网页
  • 📋 文件系统 — 读写本地文件
  • 💬 多轮交互 — 复杂任务分步执行

⚡ 一句话总结

Hermes 是 OpenClaw 的 「操作系统」

它把 AI 从聊天框里解放出来，
让 Agent 真正能动手干活

ChatGPT → 只会说话的知识分子
Hermes → 既能说话又能动手的工程师

Hermes 的典型能力

🔧 代码执行

在沙箱中运行 Python/Node.js，安装依赖，处理数据

🌐 网页访问

抓取网页内容，提交表单，API 调用，数据采集

📁 文件操作

读写文件、搜索代码库、管理项目结构

🖼️ 视觉理解

分析 UI 截图、流程图、手绘图 → 生成代码

开发实操技巧

用好 AI 的「长板」，避开 AI 的「短板」

AI 的短板：视觉稿还原

🎨 视觉稿 → 代码

AI 目前最大的弱项之一

  • ❌ 像素级还原困难
  • ❌ 多图层动效理解差
  • ❌ 设计规范难保持
  • ❌ 交互逻辑理解有限

🔍 现有方案对比

  • MaaS 应用 — 简单截图生成代码，精度有限
  • Pixso — Figma 替代品，设计稿转代码
  • 视觉大模型 — GPT-4V 等理解 UI
  • Workbuddy / 马维斯 — 专门工具

💡 当前最优实践

截图 + 结构描述 + 组件化拆分 → AI 辅助生成组件骨架，人工微调样式细节

AI 的强项：逻辑性功能

📌 场景 A：确定方案

直接告知 AI 执行即可

  • ✅ "用 ref 实现..."、"改成..."
  • ✅ 有明确技术选型
  • ✅ 有规范的标准操作

→ "把这个表格改成虚拟滚动"

📌 场景 B：不确定方案 分两轮 🎯

  • 第 1 轮：描述问题，让 AI 推荐方案
  • → 评估各方案优缺点
  • 第 2 轮：选定方案，让 AI 执行
  • → 带着明确指令干活

→ "我想实现实时协作，有什么方案？"
→ "选 OT 方案，实现基础框架"
